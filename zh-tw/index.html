瀏覽器指紋偵測器

按下方按鈕，讀取你的瀏覽器在不知不覺中暴露的指紋訊號——canvas、WebGL、已安裝字型、時區、螢幕與 User-Agent——並算出一個合成指紋 ID，再看每個訊號有多容易把你單獨認出來。所有計算都在你本機瀏覽器內完成，不會上傳任何資料。

本工具完全在你的瀏覽器本機執行，不上傳、不儲存任何資料。

瀏覽器指紋由哪些訊號組成

網站不需要 cookie，光憑瀏覽器主動回答的一堆「無害」問題，就能拼出一個相當穩定的識別碼。常見維度包括：算繪 canvas/WebGL 時顯示卡與驅動的細微差異、已安裝字型清單、時區、螢幕解析度與色深、語言、CPU 核心數、裝置記憶體、是否觸控等。單看每一項都不唯一，但組合在一起往往就唯一了。

每個訊號有多「認人」（熵）

「熵」衡量一個訊號能提供多少識別資訊（單位 bit，越高越能區分人）。EFF 的 Panopticlick 早期研究測得：在參與瀏覽器中平均約 18.1 bit、約 83.6% 的瀏覽器指紋是唯一的；AmIUnique（Laperdrix 等，2016）在 11.8 萬樣本上得到約 89.4% 唯一。下表是綜合這些公開研究、按維度給出的粗略估算。

常見問題

什麼是瀏覽器指紋？

瀏覽器指紋是把你瀏覽器與裝置主動暴露的一組屬性（UA、螢幕、時區、字型、canvas/WebGL 算繪特徵等）組合後得到的識別碼。即使不用 cookie，網站也能用它在多次造訪間識別同一個瀏覽器。

這個偵測器安全嗎？會上傳我的資料嗎？

不會。所有讀取與計算都在你本機瀏覽器以 JavaScript 完成，結果只顯示在本頁面，不向任何伺服器傳送，也不儲存。

為什麼我的指紋是唯一的？

單一屬性（如時區）很多人相同，但十幾個屬性組合後取值空間極大。公開研究顯示大多數瀏覽器的組合指紋是唯一的，因此無需登入也能被跨站追蹤。

canvas 指紋是什麼？

讓瀏覽器把同一段文字／圖形畫進隱藏 canvas，再讀出像素。不同 GPU、驅動、系統字型與抗鋸齒會產生細微差異，這個差異相當穩定，是識別力很強的一個維度。

無痕／隱私模式能隱藏指紋嗎？

基本不能。無痕模式主要清理 cookie 與本機歷史，但 canvas、WebGL、字型、時區等指紋訊號大多照舊暴露，所以單靠無痕模式無法擺脫指紋追蹤。

怎樣降低被指紋識別的風險？

可減少暴露面（統一常見設定、停用部分 API），或使用為多帳號設計的指紋瀏覽器——它給每個瀏覽環境分配一套獨立、可控且自洽的指紋，使各帳號互不關聯。

VPN 能改變我的指紋嗎？

不能。VPN 只更換出口 IP，並不改變 canvas、字型、時區、UA 等瀏覽器層訊號，因此換了 IP 指紋依舊。
